ОПЕРАЦІЙНІ СИСТЕМИ РЕАЛЬНОГО ЧАСУ ТА 32 БІТНІ МК

Лабораторна робота №7: Налагодження середовища розробки для програмування
32-розрядних мікроконтролерів з використанням операційної системи реального часу. На
прикладі FreeRTOS. 

Обладнання: STM32H743VIT6, CubeMX, STM32CubeIDE.

Мета: Ознайомитися з принципами роботи операційної системи реального часу
FreeRTOS, навчитися створювати та керувати потоками для керування світлодіодами на
STM32F411 Discovery

Завдання до лабораторної роботи:

Варіант 7. Створити три потоки для керування світлодіодами PD12, PD13, PD14. Частоти: PD12 — 1 Гц 
(тривалість 400 мс), PD13 — 0.25 Гц (тривалість 1 с), PD14 — 0.1 Гц (тривалість 2 с). Кнопка 
PA0 змінює тривалість імпульсу PD12 від 400 мс до 1 с.

Загальний опис функціоналу 💡

Програма реалізує багатозадачне керування периферією мікроконтролера на базі операційної системи реального часу FreeRTOS. Основний цикл процесора while(1) залишається порожнім, оскільки весь процесорний час динамічно розподіляється планувальником задач (Task Scheduler) між незалежними потоками.

Логіка роботи алгоритму:
- Багатозадачність: Програма ініціалізує 4 незалежні потоки (Tasks). Три потоки відповідають за асинхронне керування світлодіодами з різними частотами, а четвертий — за обробку натискань кнопки. Потоку обробки кнопки (ButtonTask) надано вищий пріоритет (2) порівняно з іншими (1) для забезпечення миттєвої реакції системи на дії користувача.
- Керування часом: Усі часові інтервали реалізовані за допомогою системної функції FreeRTOS vTaskDelay(). На відміну від блокуючої HAL_Delay(), вона переводить поточну задачу в стан очікування (Blocked state), звільняючи процесор для виконання інших паралельних задач.
- Генерація частот (Потоки індикації):
  1) Канал PD12 (1 Гц): Загальний період становить 1000 мс. Базова тривалість імпульсу (світіння) становить 400 мс, а паузи — 600 мс.
  2) Канал PD13 (0.25 Гц): Загальний період становить 4000 мс. Тривалість світіння — 1 с (1000 мс), тривалість паузи — 3 с (3000 мс).
  3) Канал PE3 (0.1 Гц): Загальний період становить 10000 мс. Тривалість світіння — 2 с (2000 мс), тривалість паузи — 8 с (8000 мс).
- Міжпроцесна взаємодія (Обробка кнопки): Тривалість світіння світлодіода на PD12 задається глобальною змінною pd12_pulse_duration. Потік кнопки постійно опитує вхід PC13.
При фіксації натискання (із реалізованим програмним антибрязкотом контактів у 300 мс) значення цієї змінної перемикається між 400 та 1000.
Потік світлодіода PD12 динамічно обчислює час світіння та залишкову паузу на основі цієї змінної. При встановленні значення 1000 мс, пауза стає рівною 0,
і світлодіод світиться безперервно протягом усього періоду.

Схема підключення ⚙️

- LED1 (PD12): GND → LED → резистор (220 Ом) → PD12. (Керується кнопкою: 400 мс або 1 с).
- LED2 (PD13): GND → LED → резистор (220 Ом) → PD13. (Блимає довго: 1 с горить, 3 с не горить).
- LED3 (PE3):  GND → LED → резистор (220 Ом) → PE13. (Блимає дуже довго: 2 с горить, 8 с не горить).
- Кнопка (PC13): VDD → Кнопка → PC13 ( з Pull-Down ).
